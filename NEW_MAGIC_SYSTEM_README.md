# 新的施法系统使用说明

## 系统概述

新的施法系统基于5.txt文档中的方案实现，具有以下特性：

1. **分阶段施法**：技能可以配置多个施法阶段，每个阶段可以显示不同的消息
2. **YAML配置**：技能数据使用YAML格式存储，支持继承和灵活配置
3. **施法打断**：角色在施法过程中受到攻击会被打断
4. **冷却系统**：技能有冷却时间限制
5. **魔法值消耗**：施放技能需要消耗魔法值

## 文件结构

```
typeclasses/
  └── magic.py              # 魔法系统核心类
world/magic/
  ├── fire.yaml             # 火系技能配置
  └── magic_index.json      # 技能索引文件
commands/
  ├── cmd_magic.py          # 新的施法命令
  └── skill_cmdset.py       # 技能命令集
```

## 技能配置格式

技能配置文件使用YAML格式，支持继承机制：

```yaml
_base:
  type: attack
  cost: 10
  cooldown: 5

fb:
  name: 火墙术
  damage:
    base: 30
    per_level: 8
  cast_time: 3
  range: 5
  stages:
    - msg: "{caster} 开始聚集火焰能量..."
      delay: 1.0
    - msg: "{caster} 双手合十，火焰在指尖跳跃..."
      delay: 1.0
    - msg: "{caster} 向{target}释放了一道炽热的火墙！"
      delay: 1.0
```

## 使用方法

1. **学习技能**：
   ```
   learn <技能名称或键名>
   ```

2. **施放技能**：
   ```
   cast <技能> <目标>
   ```
   例如：
   ```
   cast fb 怪物
   cast 火墙术 怪物
   ```

## 技能键名格式

新的施法系统支持两种技能键名格式：
1. `group>id` 格式：如 `fire>fb`
2. 简写格式：如 `fb`（系统会自动查找完整键名）

## 扩展新的技能类型

要添加新的技能类型：

1. 在 `typeclasses/magic.py` 中创建新的技能类，继承自 `SpellBase`
2. 在 `_TYPE_MAP` 中注册新的技能类型
3. 在YAML配置文件中设置技能的 `type` 字段

## 扩展新的技能组

要添加新的技能组：

1. 在 `world/magic/` 目录下创建新的YAML文件，如 `ice.yaml`
2. 在 `world/magic/magic_index.json` 中添加技能键名到文件的映射

## 系统特性

### 分阶段施法
技能可以配置多个施法阶段，每个阶段可以显示不同的消息，并设置延迟时间。

### 施法打断
当角色在施法过程中受到攻击时，施法会被打断，系统会自动清除施法状态。

### 冷却系统
技能施放后会进入冷却状态，在冷却时间内无法再次施放。

### 魔法值消耗
施放技能需要消耗魔法值，魔法值不足时无法施放技能。
